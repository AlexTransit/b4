# Основные настройки

## Веб-сервер

| Параметр | Описание | По умолчанию |
| --- | --- | --- |
| Адрес привязки | IP-адрес, на котором слушает веб-интерфейс | `0.0.0.0` (все интерфейсы) |
| Порт | Порт веб-интерфейса | `7000` |
| TLS Сертификат | Путь к файлу сертификата (для HTTPS) | — |
| TLS Ключ | Путь к файлу ключа (для HTTPS) | — |
| Язык | Язык интерфейса | Русский |

## Очередь и обработка пакетов

| Параметр | Описание | По умолчанию |
| --- | --- | --- |
| Начальный номер очереди | Номер NFQUEUE (0–65535). Изменяйте, если другие программы используют те же номера | `0` |
| Метка пакета | Метка для пакетов в iptables/nftables | `32768` |
| Рабочие потоки | Количество параллельных воркеров обработки пакетов (1–16) | `4` |
| Лимит TCP пакетов | Максимальный размер TCP-данных для анализа на соединение (байт) | `19` |
| Лимит UDP пакетов | Максимальный размер UDP-данных для анализа (байт) | `8` |

## Функции

### Протоколы

- **Поддержка IPv4** — обработка IPv4-трафика
- **Поддержка IPv6** — обработка IPv6-трафика

### Фаервол

| Параметр | Описание | По умолчанию |
| --- | --- | --- |
| Пропустить настройку IPTables/NFTables | b4 не будет создавать правила firewall (настройте вручную) | Выкл |
| Интервал мониторинга фаервола | Как часто проверять и восстанавливать правила (сек). 0 = отключён | `10` |
| Движок фаервола | Auto / nftables / iptables / iptables-legacy | Auto |
| NAT Masquerade | Включить NAT-маскарад для перенаправленного трафика | Выкл |
| Интерфейс Masquerade | На каком интерфейсе включить маскарад. Пусто = все | — |

### Сетевые интерфейсы

Выбор интерфейсов для мониторинга. Если не выбрано ни одного — мониторятся все.

| --- | --- | --- |
| Уровень логирования | ERROR / INFO / TRACE / DEBUG | INFO |
| Путь к файлу ошибок | Записывать ошибки в файл | — |

## SOCKS5 Прокси

Встроенный SOCKS5-прокси. Приложения, поддерживающие прокси, могут направлять трафик через b4 без настройки на уровне системы.

| Параметр | Описание | По умолчанию |
| --- | --- | --- |
| Включить | Запуск SOCKS5-сервера | Выкл |
| Адрес привязки | IP для прослушивания | `0.0.0.0` |
| Порт | Порт прокси | `1080` |
| Имя пользователя | Для авторизации (необязательно) | — |
| Пароль | Для авторизации (необязательно) | — |

Требует перезапуска сервиса при изменении.

## MTProto Прокси

Встроенный Telegram MTProto-прокси с fake-TLS обфускацией.

| Параметр | Описание | По умолчанию |
| --- | --- | --- |
| Включить | Запуск MTProto-сервера | Выкл |
| Адрес привязки | IP для прослушивания | `0.0.0.0` |
| Порт | Порт прокси | `3128` |
| Домен для Fake SNI | Домен, под который маскируется трафик | `storage.googleapis.com` |
| DC Relay | Адрес relay-сервера (host:port) для сценария с роутером | — |
| Secret | Секрет для подключения Telegram-клиента | Генерируется |

Требует перезапуска сервиса при изменении.

## Глобальный MSS Clamping

Ограничение TCP Maximum Segment Size. Заставляет все соединения использовать меньший размер сегмента, что приводит к естественной фрагментации трафика.

| Параметр | Описание | По умолчанию |
| --- | --- | --- |
| Включить | Активировать глобальный MSS Clamping | Выкл |
| Размер MSS | Размер MSS в байтах (10–1460) | `88` |

## Фильтрация устройств

Ограничение работы b4 трафиком от определённых устройств в сети (по MAC-адресу).

| Параметр | Описание |
| --- | --- |
| Включить | Активировать фильтрацию |
| Определение производителя | Определять vendor устройства по MAC (~6 МБ база) |
| Инвертировать выбор | Вкл = чёрный список (блокировать выбранные), Выкл = белый список (только выбранные) |

В таблице устройств для каждого можно задать:

- **Имя** — псевдоним устройства
- **MSS Clamping** — индивидуальный размер MSS (10–1460)

## Управление

Кнопки в верхней части настроек:

- **Перезапустить сервис** — перезапуск b4
- **Сбросить конфигурацию** — восстановление настроек по умолчанию (домены и категории GeoSite сохраняются)
